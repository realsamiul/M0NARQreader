The M0NARQ Pipeline

The Pipeline in Action: From a 12-Page PDF to Validated JSON in 15 Seconds.

Our architecture is not theoretical. Here is the step-by-step data flow for a 12-page bank Account Opening Form (AOF) with handwritten sections and NID card attachments—a process that manually takes 50 minutes.

See Authoritative Tech Stack
The 12-Page Form Processing Flow
01
Ingestion & Triage
Bank officer uploads a 12-page PDF. Instead of slow page-by-page OCR, our router sends the entire file to IBM's Docling for 30x faster structural parsing and page splitting.
02
Specialist VLM Routing (The "Eyes")
The router sends each page to the optimal VLM in parallel: Qwen2.5-VL reads the printed NID card barcode, Claude 3.5 reads the handwritten declaration, and Nemotron Nano parses the financial statement table.
03
Bengali LLM Processing (The "Brain")
All raw text is passed to the "Brain." TituLLM-3B parses the stated income "পঞ্চাশ হাজার টাকা" to 50000. BanglaBERT normalizes the applicant's name "মোঃ Karim আলী" for validation.
04
Cross-Validation & Fraud Check
The Qwen3-235B reasoning engine performs the final check: Does AOF.name match NID.name? ✅ Is Age (from DOB) ≥ 18? ✅ Does Stated Income match the Financial Statement? ⚠️ Flag: Anomaly Detected.
05
Final Output: Structured JSON
A single, structured JSON is generated for the bank's core banking system. It contains all validated fields, confidence scores, and a clear fraud flag with an auditable explanation. Time: 15 seconds.
